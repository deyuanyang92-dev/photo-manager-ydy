## A. 编号 与 激活
- ✅ 新建编号：地区/样地/人员/坐标 从项目继承（极少手敲）。保存按钮存全部（命名+metadata）。
- ✅ metadata 坐标继承优先级：项目默认 < 站位采集记录 < 用户手动。
- ✅ 激活编号 = "当前在拍的号"，全局唯一，是 JPG 自动归属的指针；激活即「拍摄中」；切号提醒。
- ✅ 「新建即激活」开关（默认关）。
- ✅ 阶段状态每编号独立（侧边栏 4 彩点 + 批次条）。

## B. 拍摄 → 合成（两种模式，**合成永远手动**）
软件无法判断哪些 JPG 该合成 → 合成由用户点。但**分组与命名尽量自动**。

### B1. 隐式主流程（不开分组工具）✅
```
激活编号 → 拍JPG(自动归属) → 主界面[合成]
   → 合成"未占用"的新JPG(已归属、未进任何组) → 新组 → 自动命名 编号-序号 → 消耗这批
再拍 → [合成] → 编号-2 …   = 按合成边界自动分组 + 自动命名
```

### B2. 显式分组工具（手动精细分角度）✅
```
开工具 → 自动绑编号(选中/激活/命名表单实时编号, 无需激活)
[新组] → 组1「角度1」, 再点→组2「角度2」(自动标号)
每组: 拖JPG进去 + 「输出TIF」可编辑命名 + ⚡合成/📎导入TIF + 📦整理 / ↩撤销
```
- ✅ 不激活也能用（用命名表单编号）。
- ✅ 新组自动标「角度N」。
- ✅ 每组「输出命名」可编辑：合成→编号-序号 / 导入TIF→TIF原名(去后缀) / 用户手输。

## C. TIF / JPG 处理
- ✅ 外部 Helicon GUI 合成的 TIF → 导入到组（保留 TIF 原名）→ 一键整理同名 ZIP。
- ✅ TIF 可手动删除（右键，带确认；无损母片不可恢复）。自动流程绝不删 TIF。
- ✅ 撤销合成 = 删该组 TIF（确认）+ JPG 解关联回自由池。
- ✅ 导入 TIF 候选扫描走配置目录（含遗留 新拍JPG）。

## D. 整理 / 归档
- ✅ 整理 = 源 JPG 打包 JXL+ZIP（与输出 TIF 同名）+ 移 results。4 道删 JPG 安全闸；TIF 不删。
- ✅ 整理前命名兜底：合成 TIF 名不符成果规范 → 弹确认框按编号成果名改名（安全网，非主路径）。
- ✅ 「合成后自动整理」开关（默认关）。
- ✅ incoming/results 子目录可配，全链路读写一致。

## E. 监控 / 归属
- ✅ 实时监听 + firstSeenAt 归属（非 mtime）。
- ✅ JPG 手动归属 / 取消（取消 = 变无主 + 踢出合成组；重归属解黑名单）。
